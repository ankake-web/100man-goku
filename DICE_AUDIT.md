# ダイス可視化 監査・実装レポート（Cities & Knights）

スコープ: 3個のダイス（赤・黄・イベント）の可視化と連動演出、および発展カード抽選条件（赤 ≤ Lv+1）の検証。
他のルール/ロジックには触れない。

## ステップ1：現状把握
- **生産ダイス**: `playDiceRoll(d1, d2)`（src/main.ts）が「白いサイコロ2個」をタンブル表示。
  ターンパネルにも `🎲 d1+d2=total` のテキストのみ。色の区別なし。
- **イベントダイスのロジックは既に内部に存在**（二重実装の懸念を確認 → 無し）:
  - `rollEventDie(rng)` / `applyEventDie(state, rng, redDie)`（src/engine/citiesKnights.ts）。
  - ROLL_DICE（game.ts）が毎ターン `applyEventDie` を1回呼び、**船=蛮族トラック前進・襲来判定**、
    **色ゲート=進歩カード抽選** を実行。結果は `state.lastEventDie`、`state.barbarianPosition` 等に格納。
  - つまり「赤＝生産d1（抽選しきい値にも使用）／黄＝生産d2／イベント＝lastEventDie」というモデルは
    内部的に成立済みだったが、**UIに一切可視化されていなかった**（直前タスクでターンパネルに文字表示と
    蛮族ピップ、襲来ログは追加済）。

## ステップ2：イベントダイスの統合方法
- 二重実装を作らず、**既存 `applyEventDie` を唯一の出所**として表面化:
  - ロール演出を2個→**3個**に拡張。第3の「イベントダイス」は `state.lastEventDie` を表示。
  - 生産ダイスを **赤(d1)／黄(d2)** に色分け（内部表現は従来どおり `lastDiceRoll=[d1,d2]`、d1=赤=抽選しきい値）。
  - 6面の内訳（船×3／青×1緑×1黄×1）は `rollEventDie` の実装どおり。タンブル中はその6面を点滅。
- `runWithDiceAnim` に **直前stateを渡し**、`buildDiceEventInfo(prev, next)` で可視化情報（イベント面・赤値・
  蛮族前進/襲来・各プレイヤーの抽選照合）を導出。船/色ゲートでパネルを出し分け。

## ステップ3：抽選条件の検証・修正
- 条件は「色ゲートの色について、赤 ≤ そのトラックLv+1」で **概ね正しかった**（直前タスクで Lv≧赤 の1ズレを修正済）。
- 今回の精査で **未改良(Lv0)が赤=1のとき誤って引けるバグ** を発見 → **修正**:
  `drawProgressCards` に `lvl >= 1` を追加（Lv0は赤に関わらず引けない。Lv1=1–2 … Lv5=1–6）。
- 併せて **引く順序を「現手番から時計回り」に修正**（従来は playerOrder[0] 起点。山札枯渇時の順序を正す）。
- 検証: フルCPU対戦で **Lv0が引いた回数=0**、抽選延べ28人・蛮族前進23・襲来3 を確認。境界テスト(A2)追加。

## ステップ4：追加した演出
- **3個ダイス**: 赤(角丸四角・白ピップ)／黄(角丸四角・黒ピップ)／イベント(**円形**・色背景＝形でも区別)。
  - イベント面の色: 船=灰、商業=黄、政治=青、科学=緑。タンブル→確定。
  - 合計表示を `赤d1 ＋ 黄d2 ＝ 生産N` に変更し、赤が生産と抽選の両方に使われることを明示。
- **イベント結果パネル**（ロール直後に大きく提示、約1.6秒）:
  - **船** → 「🛶 蛮族船が前進」/「⚔ 蛮族 襲来！」＋蛮族トラックのピップ（前進マスを強調・残り2マスで赤警告）。
  - **色ゲート** → 「{色} の抽選（赤=N ≤ Lv+1?）」＋各プレイヤーを**手番順**に「Lv／しきい値（赤≤Lv+1）／結果」で照合表示。
    引ける人をハイライト、未改良/未達は簡潔表示、実際に引いた人は「🎴 +1」。
- **常時表示**: ターンパネルに 赤/黄/イベント の色チップを常設（オーバーレイ消滅後も結果が残る）。
  蛮族トラックのピップ（n/7・残り2マスで警告色「まもなく襲来」）も常時表示（直前タスク）。
- **7のとき**: 生産なし。イベントパネル（蛮族前進 or 色抽選）を見せた後、**既存の7処理**（手札破棄・初回襲来前は盗賊凍結）に
  そのまま接続。7処理のロジックは不変。

## 残課題
1. **蛮族襲来の対決ビュー**（防衛力 数値 vs 蛮族強度 の対比アニメ）は未。今回は「襲来！」表示＋既存の襲来ログ（誰が守護VP/都市略奪）で代替。
2. **抽選のフリップ演出**はパネル内のテキスト（🎴 +1・手番順の行）で簡易表現。山札から1枚めくる物理アニメは未。
3. イベントダイスの面は記号（🛶/商/政/科）表示。専用アイコン画像化は任意。
4. モーション抑制/最速設定では従来どおり演出を簡略化（最速は完全スキップ）。
