|--------|---------|----------------|

| 鉄鉱石 | `ore` | 19 |

> **バンク枯渇ルール**: バンクの在庫が不足している場合、**在庫分を上限として渡す**（例: バンクに wood が1枚しかなく2枚産出するはずなら1枚だけ渡す）。在庫が0枚の場合のみ産出0枚となる。全か無かルールは適用しない。  
> **実装注**: バンク枚数はGameStateで管理。産出計算時に `Math.min(産出量, bank.stock[resource])` を適用し、実際に渡した枚数をログに残す。

## 5. 発展カード（28枚）

### 5-1. カード内訳

| カード名 | 英語キー | 枚数 | 種別 |
|---------|---------|------|------|
| 騎士（Knight） | `knight` | 14 | アクション |
| 道路建設（Road Building） | `road_building` | 2 | アクション |
| 年の豊穣（Year of Plenty） | `year_of_plenty` | 2 | アクション |
| 独占（Monopoly） | `monopoly` | 2 | アクション |
| 勝利点（Victory Point） | `victory_point` | 5 | 常時効果 |

### 5-2. 使用ルール

- 購入したターンには使用不可（次のターン以降に使用可能）。
- 1ターンに使用できるアクションカードは**1枚のみ**。
- 勝利点カードは手札に保持し続け、VP10到達宣言時に公開する。
- 使用済みカードは「使用済みデッキ（discard pile）」へ。

**「購入ターン判定」の実装方針（Logi向け）**:

```typescript
// GameState に以下フィールドを追加
interface DevCard {
  id: string;
  type: DevCardType;
  purchasedOnTurn: number;  // 購入したターン番号を記録
}

// 使用可否チェック
canPlay(card: DevCard, currentTurn: number): boolean {
  return card.purchasedOnTurn < currentTurn;
}
```

> ターン番号はゲーム開始から単調増加する整数。プレイヤーごとのターン番号ではなく、**ゲーム全体のターン番号**を使用すること（4人の場合、1ターン＝4手番分）。

### 5-3. 各カード効果

| カード | 効果 |
|--------|------|
| 騎士 | 強盗（Robber）を任意のタイルへ移動、隣接プレイヤーから1枚盗む |
| 道路建設 | 無償で道を2本建設（資源不要。ただし通常の建設制約に従う） |
| 年の豊穣 | バンクから任意の資源を2枚受け取る（異なる種類でも可） |
| 独占 | 資源1種を宣言、全プレイヤーがその資源を全て渡す |
| 勝利点 | 保持中ずっと+1VP（公開不要） |

**道路建設カードの部分建設ルール（R-05）**:

- 道を2本建設できる状態でのみカードを使用可能とする（使用前に建設可能頂点が1箇所以上存在するかチェック）。
- ただし、1本目を置いた後に盤面が詰まり2本目が置けない場合は、**1本のみ建設して終了**とする（カードは消費済み扱い）。
- 「2本置けないなら使用不可」とはしない。1本でも置ける状況ならカードは使用可能。

**独占カードの0枚ケース（R-09）**:

- 宣言した資源を全プレイヤーが1枚も持っていない場合、カードは使用済みになるが何も起きない（資源の移動なし）。
- UIには「誰も〇〇を持っていませんでした」と表示する。

## 6. 強盗（Robber）

### 6-1. 発動条件

- ダイス目が**7**の場合、強盗フェーズへ移行。
- 騎士カード使用時も同じフローを実行。

### 6-2. 処理手順

1. **手札上限チェック**: 手札が**8枚以上**の全プレイヤーは、手札の**半数（切り捨て）**を捨てる。
2. **Robber移動**: アクティブプレイヤーが砂漠以外の任意タイルへRobberを移動。
3. **盗み**: 移動先タイルに隣接する開拓地/都市を持つ**他プレイヤー**から、1人を選んでランダムに1枚盗む。

> **注**: 移動先タイルに自分の開拓地/都市しかない場合、または他プレイヤーが隣接していない場合は盗みをスキップ。

## 7. 勝利点（Victory Point）計算

### 7-1. 点数テーブル

| 要素 | VP |
|------|----|
| 開拓地 1個 | +1 |
| 都市 1個 | +2 |
| 最長道路ボーナス | +2 |
| 最大騎士団ボーナス | +2 |
| 勝利点カード 1枚 | +1 |

### 7-2. 最長道路（Longest Road）

- **最低条件**: 連続した道が**5本以上**。
- 最長の連続道路を持つプレイヤーが「最長道路カード」（+2VP）を保持。
- 他プレイヤーがそれを上回ったとき、カードは即座に移動する。
- **連続道路の切断ルール**:
  - **相手プレイヤー**の開拓地/都市が頂点に存在する場合、そこで道路は切断される。
  - **自分自身**の開拓地/都市は切断しない（自分のコマがあっても道路は繋がり続ける）。
- **計算アルゴリズム方針（Logi向け）**: 対象プレイヤーの道Edgeのみを対象とした無向グラフを構築し、各Edgeを起点にDFS（深さ優先探索）で到達可能な最長パスを求める。分岐は全経路を探索し最長値を採用。計算量は最大O(E!)だが道の上限15本のため許容範囲。

### 7-3. 最大騎士団（Largest Army）

- **最低条件**: 騎士カードを**3枚以上**使用。
- 最も多くの騎士カードを使用したプレイヤーが保持。
- 同数の場合、現保持者が優先（後からの同数では移らない）。

### 7-4. 勝利宣言タイミング

- 自分のターン中であれば、**いつでも**（ダイスロール前でも、建設後でも）宣言可能。
- 相手のターン中は宣言不可（勝利点カードのVPで10点を超えていても待つ）。

**勝利チェックのトリガーポイント（R-08）（Logi・Fenn向け）**:

以下のいずれかのアクションが完了した直後に `checkVictory(activePlayer)` を呼び出す。

| トリガー | 変化するVP要素 |
|---------|-------------|
| 開拓地建設完了 | +1VP |
| 都市建設完了 | +1VP（開拓地→都市で差分）|
| 発展カード購入完了 | 勝利点カードの場合+1VP（即時加算）|
| 発展カード使用完了 | 騎士→最大騎士団変動の可能性 |
| 道建設完了 | 最長道路変動の可能性 |
| ターン開始時（ダイスロール前） | 前ターンの変動を念のため再計算 |

> **注**: 相手プレイヤーのアクション（騎士カードで最大騎士団が移動、最長道路が移動）によって自分のVPが変動することがある。その場合でも勝利チェックは**次の自分のターン**まで行わない。`checkVictory` はアクティブプレイヤーのみ対象とする。

## 8. 初期配置

### 8-1. 順序

1. スタートプレイヤーから時計回りに開拓地1個＋道1本を配置。
2. 最後のプレイヤーからの逆順（スネーク方式）で2個目の開拓地＋道1本を配置。
3. 2個目の開拓地に隣接するタイルから初期資源を受け取る（1個目からは受け取らない）。

### 8-2. 配置制約

- 開拓地は他の開拓地/都市から**2頂点以上**離れなければならない（距離ルール）。
- 道は自分の開拓地/都市または既存の道に接続しなければならない。
