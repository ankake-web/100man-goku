# 航海者（Seafarers）拡張 実装プラン

> 進捗: **Phase 0（土台）+ 1（船）+ 3（島ボーナスVP）+ 5（基本AIの船活用）実装済み**。
> ホームで「盤面（ルール）: 基本／航海者」を選択でき、CPU 対戦で CPU 自身が海峡を渡って
> 新島へ入植し、最初の入植者には +2VP が入る（島の核となる目標が機能する）。
> 未実装（後続）: 金タイルの産出（Phase 2・現マップは金タイル未配置）/ 最長交易路への船算入 /
> 海賊・船移動（Phase 4）/ オンライン対戦でのシナリオ選択（Phase 6）。


カタン拡張「航海者（Seafarers）」を、**基本ゲームを壊さずに**段階的に追加するためのロードマップ。
各フェーズは可能な限り「単体で検証できる」単位に分割する。最優先は既存機能の非破壊
（PROGRESS.md の方針に従う）。基本ゲームは `scenario: 'classic'` として温存する。

## 設計方針

- **シナリオ駆動**: 盤面生成・特別ルール・勝利条件を「シナリオ」として抽象化する。
  `classic`（基本）を既定シナリオとし、`seafarers_*` を追加する。
- **エンジンは純粋関数のまま**: `applyAction(state, action)` リデューサ方式を維持。
  新アクション（`BUILD_SHIP` 等）も同経路で追加し、UI から独立してテストする。
- **可変盤**: `buildBoardGeometry(coords)` を座標集合パラメータ化（既定＝従来の19タイル）。
  海タイルを含む大きめ/島構成の盤を生成できるようにする。
- **LAN 対応は最後**: まず CPU 対戦で遊べる状態を作り、その後 LAN（マスク/プロトコル/
  サーバ検証）へ広げる。

## 新しい盤面要素

- **海タイル (`sea`)**: 数字なし・資源を産出しない。海に面した頂点には沿岸開拓地を置けるが、
  海タイルだけに囲まれた内陸頂点には置けない。
- **金タイル (`gold`)**: 数字あり。出目一致時、所有者は**任意の資源**を選んで得る
  （開拓地=1・都市=2）。
- **船 (ship)**: 道とは別の建設物。海に面した辺（sea-edge）に置く。コスト=木+羊。
  自分の交易路を海上に延ばす。道と船は**自分の開拓地/都市でのみ接続**を切り替えられる。
- **最長交易路**: 道＋船の連続経路で計算（道↔船の切替は建物経由のみ）。
- **海賊 (pirate)**: 海版の盗賊。7や騎士で海タイルへ移動し、隣接する船の所有者から奪う／
  そのタイル隣接の船建設を封じる。（後半フェーズ）

## フェーズ

### Phase 0 — 土台（非破壊） ← 最初に実装
- `TileType` に `'sea' | 'gold'` を追加。
- `buildBoardGeometry(coords?)` / `createEmptyTiles(coords?)` を座標集合パラメータ化（既定＝classic）。
- `src/engine/scenarios.ts`: シナリオ登録。`classic` と最初の航海者マップ
  `seafarers_newshores`（新たな海岸を求めて・簡易版）の盤面生成（geometry＋tiles＋harbors）。
- `createInitialGameState(..., scenario?)` にシナリオ引数（既定 `classic`、非破壊）。
- 描画: `sea`（波の水タイル・数字なし）/`gold`（金タイル・数字あり）を renderer に追加。
- 検証: 既存テスト全緑（classic 不変）＋ 航海者マップ生成のユニットテスト＋ 盤面スクショ。
- この段階では**まだ遊べない**（船が無い）。UI からは選べないままにする。

### Phase 1 — 船と海上交易路（航海者の中核）
- `Edge.ship`（または `road | ship` 排他）を追加。`Player.remainingShips`（初期15）。
- `BUILD_SHIP` アクション: コスト木+羊、sea-edge 限定、接続規則（建物 or 自分の船に連結）。
- 配置バリデーション: sea-edge=海タイルに面する辺。道は純海辺に置けない／船は sea-edge のみ。
- 最長交易路 DFS を道＋船対応に拡張（建物での切替規則）。
- 描画: 船を道と区別（ボート表現）。候補ハイライト。
- セットアップ: 航海者では初期配置の「道」を、海に面していれば船で置けるように。
- テスト: 接続規則・最長交易路・コストの回帰テスト。

### Phase 2 — 金タイルの産出
- 金タイルの出目一致時、所有者が任意資源を選ぶUI（年の豊穣に類似）。開拓地1/都市2。
- 銀行在庫・複数人同時の処理。CPU は不足資源を選ぶ。

### Phase 3 — シナリオ勝利条件と島 ✅ 実装済み
- 「新たな海岸を求めて」: ホーム島以外の島に**新規開拓地**を建てると +2VP（最初の入植者）。
- `src/engine/islands.ts`（海以外タイルの連結成分＝島）＋ `GameState.islandBonus`。
  MAIN の `BUILD_SETTLEMENT` で「島内の建物が初」のとき付与し calcVP に算入。
  基本ゲームは海タイルが無く全タイル1島のため常に no-op（非破壊）。
- VP 目標は標準の 10 のまま（島ボーナスは到達を早める。目標自体の変更は別途バランス判断）。

### Phase 4 — 船の移動・海賊（上級ルール）
- 交易路の開放端の船を1ターン1回移動（航海）。UI＋規則。
- 海賊: 海タイル用の第2の盗賊。7時に盗賊/海賊を選択。隣接船から奪取・建設封鎖。

### Phase 5 — AI（航海者対応） ✅ 基本実装済み（船・島拡張）
- CPU が船で海峡を渡り新島へ入植する（`ai.ts bestExpansionShip`＝海辺の Dijkstra で
  「価値 − ペナルティ×隻数」最大の経路の次の1隻を選ぶ）。普通/強に組込み、弱・基本は不変。
- 残り（後続）: 金タイルの資源選択（Phase 2 と同時）・海賊（Phase 4 と同時）。

### Phase 6 — オンライン対戦（LAN）対応
- 船・金選択・海賊の同期（mask / protocol / サーバ検証）。新アクションを許可リストへ。
- ロビーでシナリオ選択をホストが設定。

### Phase 7 — 仕上げ
- 演出（船設置・航海）、効果音、モバイル最適化、バランス調整、ドキュメント。

## 最初の「遊べる」マイルストーン（CPU対戦）
Phase 0 + 1 + 2 + 3 + 5（基本AI）で、海賊・船移動・LAN を後回しにした
**「新たな海岸を求めて（簡易）」を CPU と遊べる**状態を目標にする。
ホーム画面に「ルール: 基本 / 航海者」セレクタを追加して切替可能にするのは Phase 1 完了後。
