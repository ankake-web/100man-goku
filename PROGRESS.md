## 2. 発見したが直さなかった問題と理由

- **`getElementById('cpu-status')?.remove()` の no-op 呼び出しが4箇所残存**：CPU状況バナーを撤去したため対象要素はもう生成されない（完全に無害な防御コード）。複数の複合クリーンアップ行に埋まっており、除去のリスク＞便益のため温存。
- **リンター/フォーマッタ未導入**：ESLint/Prettier の設定もスクリプトも無い。「完成一歩手前」での新規ツール導入は方針（設計/ツール変更を避ける）に反するため**追加しない**。代わりに `tsc --strict` ＋ 一回限りの `noUnusedLocals` チェックで死蔵コードを掃除済み。導入可否は人間判断（下記）。
- **難易度バランス**：AI はヒューリスティックで調整済み。明らかな破綻は観測されず、仕様変更にあたる調整はしていない（実プレイでの体感確認を推奨）。
- **OGP の `og:image`**：ホスト済み画像アセットが無いため未設定（タイトル/説明の OGP のみ追加）。共有時のサムネ画像が必要なら画像を用意して追加。

## 3. 人間の判断が必要なリスト

1. **リンター/フォーマッタの導入可否**：現状なし。導入するなら別途セットアップ作業（依存追加・設定・CI 連携）が必要。今回は方針に従い見送り。
2. **`og:image`（SNS 共有サムネ）**：必要なら 1200×630 程度の画像を `public/` に置き、`index.html` に `og:image`/`twitter:card` を追加。
3. **クロスブラウザ目視**：自動確認は Chromium のみ。Safari / Firefox（特に iOS Safari）でのレイアウト・タッチ操作・`background-clip:text` のタイトル表示・ピンチズームは**実機目視が必要**。
4. **オンライン対戦の本番疎通**：サーバ側の不正メッセージ耐性は修正したが、Render 上での実デプロイ・`ALLOWED_ORIGINS` 設定・2端末同時接続の通し確認は実環境で。
5. **難易度・テンポの体感**：CPU の強さ/速度が「競るが理不尽でない」かは実プレイで最終判断を。
