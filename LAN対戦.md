# LAN対戦の遊び方（実機確認手順）

同じ Wi-Fi / 同一LAN内の複数端末（PC・スマホ）で、**人間同士**または**人間＋CPU**のカタン対戦ができます。
Firebase や外部クラウドは使いません。ホストPCの開発サーバ上で完結します。

対応構成（合計2〜4人 / 人間1〜4人 / CPU 0〜3人。CPUだけの対戦は不可）:
人間2/3/4人のみ、人間1人＋CPU1〜3人、人間2人＋CPU1〜2人、人間3人＋CPU1人。

---

## 1. 事前準備

- ホストにするPCと、参加する端末（スマホ/別PC）を **同じ Wi-Fi（同一LAN）** に接続する。
- 依存関係の用意（初回のみ）:
  ```sh
  cd ~/games/catan
  npm install
  ```

## 2. ホストPCでサーバを起動（LAN公開）

```sh
cd ~/games/catan
npm run dev:lan
```

起動すると次のように表示されます（`Network:` がLAN公開URL）:

```
  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

- **ホストURL** = `Network:` の行（例 `http://192.168.101.96:5173/`）。
- IPアドレス部分はネットワークによって変わります。表示された値を使ってください。
- 通常の `npm run dev` は localhost のみで、他端末からは見えません。**LAN対戦時は `npm run dev:lan`** を使ってください。

> ファイアウォールでポート 5173 の受信がブロックされている場合は許可してください
> （macOS: システム設定 → ネットワーク → ファイアウォール）。

## 3. ホスト（PC）でルームを作成

1. ブラウザで `http://localhost:5173/`（PC自身）を開く。
2. 上部タブの **「オンライン対戦」** を選ぶ。
3. プレイヤー名を入力し、**「ルームを作成（ホストになる）」** を押す。
4. 画面に **ルームコード（4文字）** と、参加用の **ホストURL** が表示されます。
   - このルームコードとURLを参加者に伝えます。

## 4. スマホ／別端末から参加

1. スマホのブラウザで、ホスト画面に表示された **ホストURL**（`http://192.168.x.x:5173/`）を開く。
2. **「オンライン対戦」** タブを選ぶ。
3. プレイヤー名を入力し、**ルームコード** を入力して **「参加」** を押す。
4. ホスト・参加者の双方に **参加者一覧** が同期表示されます（2〜4人まで）。

## 4.5. CPU を混ぜる（人間＋CPU 対戦）

ホストのロビーに **「CPU 人数」** の `−` / `＋` ボタンがあります（ホストのみ操作可）。

- `＋` / `−` で CPU を 0〜3人に設定します。**人間＋CPU が合計2〜4人**になるよう自動調整されます。
- 参加者一覧に「🤖 CPU α / CPU β / CPU γ」が表示されます。
- 人間が増えると、上限を超えるぶんの CPU 枠は自動で減ります。
- 例:
  - **人間1人＋CPU1〜3人**: 1人で作成し、CPU を 1〜3 に設定して開始。
  - **人間2人のみ**: 2人参加・CPU 0 で開始。
  - **人間2人＋CPU1人**: 2人参加・CPU を 1 に設定して開始。
  - **人間2人＋CPU2人**: 2人参加・CPU を 2 に設定して開始。
  - **人間3人のみ / 人間3人＋CPU1人**: 3人参加・CPU 0 または 1 で開始。
  - **人間4人のみ**: 4人参加・CPU 0 で開始。
- CPU はすべて **ホストPCのサーバ側** で自動的に動きます（手札内容は誰にも見えません）。

## 5. ゲーム開始〜プレイ

1. **人間1人以上・合計2〜4人**になったら、**ホストの画面で「ゲーム開始」** を押す。
2. 全端末に同じ盤面が表示されます。各端末には自分のプレイヤー色・手番順が割り当てられます（CPU も手番順に入ります）。
3. **手番の進め方**
   - 初期配置: 手番の人の端末だけ盤面に配置候補がハイライトされます。タップして開拓地→道を置きます。順番に全員ぶん行います。
   - 通常ターン: 手番の人が「🎲 ダイスを振る」→ 資源配布 → 建設/交易 → 「↩ ターン終了」。
   - 手番でない端末には「⏳ ○○ の操作を待っています…」と表示され、操作ボタンは出ません。
   - **CPU の番**は「🤖 CPU α が操作中です…」と表示され、サーバ側で自動進行します（人間は待つだけ）。CPU の建設・ダイス・盗賊などはログに表示されます。
4. **対応している操作（全端末同期）**
   - サイコロ、資源配布、道/開拓地/都市の建設、発展カード購入
   - プレイヤー間交易（提案→対象が承認/拒否→提案者が成立）、銀行交易
   - 7（捨て札→各自の端末で半分捨てる）、盗賊移動・奪取（サーバ側でランダム）
   - 発展カード使用（騎士・街道建設・年の豊穣・独占）、勝利判定・勝利演出
5. **手札の見え方**
   - 自分の資源・発展カードは内容まで見えます。
   - 他プレイヤーは「枚数」だけ表示され、内容は見えません（受信データ上も秘匿）。

## 6. 確認できること（実機チェックリスト）

- [ ] PCでルーム作成、スマホからホストURLにアクセス・参加できる
- [ ] 参加者一覧が全端末で同期する
- [ ] ゲーム開始で全端末に同じ盤面が出る
- [ ] 初期配置が手番順に進む（手番でない端末は配置不可）
- [ ] サイコロの出目が全端末で一致し、資源配布が同期する
- [ ] 道/開拓地/都市の建設、ターン終了が同期する
- [ ] 交易の提案→承認/拒否→成立が同期する（承認/拒否は対象の端末だけ）
- [ ] 7/捨て札/盗賊が破綻しない（CPU が対象でも自動で処理される）
- [ ] 発展カード使用が同期する
- [ ] 自分の手札だけ内容が見え、他人／CPU は枚数のみ
- [ ] CPU 人数を設定して開始できる（人間＋CPU 合計2〜4人）
- [ ] CPU の番が自動で進み、ログに CPU の行動が出る
- [ ] スマホ幅で横スクロールや見切れがない

## 7. トラブルシューティング

| 症状 | 対処 |
|---|---|
| スマホからホストURLが開けない | 同じ Wi-Fi か確認 / PCのファイアウォールでポート5173の受信を許可 / `npm run dev:lan` で起動しているか確認 / VPN・ゲストネットワーク・会社ネットワークだと端末間通信が遮断される場合あり |
| 「サーバに接続できませんでした」 | ホストPCのサーバが起動中か、URL（IP:ポート）が正しいか確認 |
| 「ルームが見つかりません」 | ルームコードの入力ミス（4文字・大文字）。ホスト画面のコードを再確認 |
| 「ルームが満員です」「合計2〜4人で開始」 | 人間＋CPU は最大4人・最小2人（人間は1人以上）。CPU 人数を調整 |
| WebSocket が切れる / 画面が更新されない | ホストPCがスリープ・Wi-Fi 切替で切断。ホストの `dev:lan` を再起動し、各端末でページを再読み込み |
| ゲーム開始後に画面が固まる | 手番の人が操作する番です（⏳表示）。**切断した人の番**だと進みません（下記制限） |
| ホストPCがスリープした | サーバが止まります。スリープしない設定にし、再起動後はやり直し |
| CPU が動かない | CPU はホストの `dev:lan` サーバが動かします。サーバ起動中か確認。`dev:lan` を再起動（WS サーバは再起動で反映） |
| 切断後に進まない | 進行中の切断・再接続は未対応（下記制限）。やり直してください |

---

## 8. 制限事項（現状）

- 開発サーバ（`vite --host`）上での対戦を想定しています。本番ビルド（`vite build`）での配信は未対応です。
- インターネット越し（別拠点）の対戦・認証・アカウントには対応していません（同一LAN内のみ）。Firebase は未使用です。
- **CPU はホストPCのサーバ側**で動きます。CPUだけの対戦はできません（人間1人以上が必要）。
- ゲーム開始後の切断/再接続は限定対応です。切断は他プレイヤーに通知されますが、**切断した人の手番になると進行が止まります**（再接続による復帰は未対応）。参加者は開始前にそろえてください。
- CPU の未対応な細かい挙動（残課題）:
  - CPU から人間への交易「提案」はしません（人間→CPU の承認/拒否は対応）。
  - CPU の発展カード使用は「騎士（ダイス前）」中心で、街道建設/年の豊穣/独占の自発使用は限定的です。
- 交易のカウンターオファー（反対提案）・タイムアウトは未対応です。
